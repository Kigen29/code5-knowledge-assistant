# Code 5 Developers Knowledge Assistant

A Flask-based Retrieval-Augmented Generation application for Code 5 Developers, a software development, systems development, maintenance, and technology services company.

## Features

- Chat UI at `/` for asking questions about Code 5 Developers documents
- Document library at `/library`
- PDF and Markdown document reader pages
- JSON chat API at `/chat`
- Health check at `/health`
- Markdown/TXT/HTML/PDF ingestion pipeline
- Large PDF manuals generated by `scripts/generate_policy_pdfs.py`
- Local Chroma vector store with Chroma's ONNX MiniLM embedding model
- Groq chat generation through `GROQ_API_KEY`
- Offline extractive fallback for demos and tests without an API key
- Evaluation script for citation accuracy, groundedness, and latency
- GitHub Actions CI and Render deployment config

## Setup

```bash
python3.11 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

Optional Groq configuration:

```bash
export GROQ_API_KEY="your-key"
export LLM_MODEL="llama-3.1-8b-instant"
```

## Build the Library and Index

```bash
python scripts/generate_policy_pdfs.py
python -m rag_app.ingest
```

This creates generated Code 5 Developers PDF manuals in `policies/` and a local Chroma index in `.chroma/`. The first run may download Chroma's local ONNX MiniLM embedding model.

## Run Locally

```bash
python -m rag_app.app
```

Open `http://127.0.0.1:5000`.

## API

```bash
curl -X POST http://127.0.0.1:5000/chat \
  -H "Content-Type: application/json" \
  -d '{"question":"What is Code 5 Developers standard software delivery workflow?"}'
```

## Test and Evaluate

```bash
pytest -q
python -m rag_app.evaluate
```

The evaluation script writes `eval/results.json` with p50/p95 latency, automatic citation checks, and a manual review note.

## Render Deployment

1. Push this repo to GitHub.
2. Create a new Render web service from the repo, or use `render.yaml`.
3. Set `GROQ_API_KEY` as a secret environment variable.
4. Use build command: `pip install -r requirements.txt && python scripts/generate_policy_pdfs.py && python -m rag_app.ingest`.
5. Use start command: `gunicorn 'rag_app.app:app' --bind 0.0.0.0:$PORT`.
6. Set health check path to `/health`.

After deployment, add the public URL to `deployed.md`.
