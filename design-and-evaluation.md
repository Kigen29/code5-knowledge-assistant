# Design and Evaluation

## Architecture

The application is a Flask web service for Code 5 Developers. It provides `/` for the knowledge assistant, `/library` for browsing documents, `/library/<filename>` for reading Markdown or PDF documents, `/documents/<filename>` for direct file access, `/chat` for JSON question answering, and `/health` for deployment monitoring.

The RAG pipeline is split into document parsing, chunking, vector indexing, retrieval, answer generation, and response formatting. Answers are shown as clean prose, while citations and snippets are displayed separately.

## Corpus

The corpus is a synthetic Code 5 Developers knowledge base. It covers software delivery, systems development, maintenance, support, DevOps, client operations, security, QA, technology procurement, onboarding, incident response, AI automation, and internal operations.

The corpus includes Markdown policies plus generated PDF manuals. The PDFs are generated by `scripts/generate_policy_pdfs.py` so they are reproducible and legally safe to include.

## Design Choices

- **Embedding model:** Chroma's local ONNX MiniLM embedding function, chosen because it is free, local, and lighter to deploy than a PyTorch `sentence-transformers` stack.
- **Vector store:** Chroma persistent local storage, chosen for simple setup and a lightweight deployment footprint.
- **Chunking:** Markdown headings are used as natural boundaries, with a word-window fallback for long sections and PDFs.
- **Retrieval:** Top-k retrieval defaults to `k=5`, balancing coverage and prompt size.
- **Generation:** Groq with `llama-3.1-8b-instant` is the default hosted LLM through `GROQ_API_KEY`. If no key is present, the app uses an extractive fallback from retrieved snippets so local demos remain functional.
- **Guardrails:** The assistant refuses out-of-corpus questions, keeps answers concise, removes inline citation markers from the answer area, and returns citations and snippets in separate UI panels.

## Evaluation Approach

The evaluation set contains Code 5 Developers questions with expected answers and supporting source titles, including questions whose evidence lives in generated PDF manuals. `python -m rag_app.evaluate` runs each question through the live pipeline and reports:

- **Groundedness auto-check:** whether the answer was supported enough to avoid the refusal response.
- **Citation accuracy auto-check:** whether returned citations include the expected document title.
- **Latency:** p50 and p95 response time in milliseconds.

Before final submission, manually review `eval/results.json` to confirm that each answer is factually supported by the retrieved snippets and that citations point to the passages that support the claim.

## Current Results

Run after regenerating PDFs and rebuilding the index:

```bash
python scripts/generate_policy_pdfs.py
python -m rag_app.ingest
python -m rag_app.evaluate
```

The latest local evaluation run produced:

- Indexed corpus: 20 Code 5 Developers documents
- Generated PDF manuals: 10 PDFs totaling 166 pages
- Indexed chunks: 176
- Question count: 30
- Groundedness auto-check: 100%
- Citation accuracy auto-check: 100%
- Latency p50: 39 ms
- Latency p95: 164 ms

Before final submission, rerun the evaluation after any document changes and manually review the generated answers against retrieved snippets.

## CI/CD

GitHub Actions installs dependencies, imports the Flask app, and runs the pytest suite on push and pull request. Render deployment builds the PDF manuals and Chroma index during the build step, starts Gunicorn, and checks `/health`.
