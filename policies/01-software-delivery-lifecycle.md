# Software Delivery Lifecycle Policy

## Purpose
Code 5 Developers builds software systems, web platforms, mobile applications, integrations, automation tools, and internal business systems for clients. This policy defines the delivery lifecycle used to move work from discovery to production support.

## Discovery and Scoping
Every project begins with discovery. The project lead must document the business goal, target users, current pain points, core workflows, success metrics, integrations, constraints, and known risks. Discovery outputs must include a scope note, stakeholder list, assumptions, exclusions, and a first release recommendation.

## Requirements Baseline
Before development starts, the team must agree on a baseline backlog. Each feature should include a user outcome, acceptance criteria, priority, dependencies, and test notes. The baseline may change, but changes must be visible to the client and approved by the project lead.

## Delivery Phases
Code 5 Developers uses iterative delivery. Standard phases are discovery, solution design, implementation, quality assurance, user acceptance testing, deployment, and support transition. Small projects may combine phases, but no project may skip testing, deployment review, or handover.

## Definition of Ready
A task is ready for development when the requirement is clear, the expected behavior is testable, designs or API contracts are available where needed, and blocking decisions have been resolved. Developers should not start vague tasks that cannot be reviewed objectively.

## Definition of Done
A task is done when code is reviewed, tests pass, acceptance criteria are met, documentation is updated, security concerns are addressed, and the work is deployed or prepared for release according to the project plan.

## Change Control
Changes that affect budget, timeline, architecture, integrations, user experience, or security must be recorded as change requests. The project lead explains the impact, recommends an option, and obtains written approval before the team commits to the change.
