# Technology Procurement Policy

## Purpose
Code 5 Developers uses third-party services, cloud platforms, developer tools, domains, plugins, APIs, and software licenses to deliver client work. This policy governs selection, approval, and management of technology purchases.

## Tool Selection
Tools should be selected based on client requirements, security, reliability, maintainability, pricing, integration fit, documentation, export options, and vendor reputation. Engineers should avoid adding tools that duplicate existing approved capabilities without a clear reason.

## Approval
New paid tools, cloud services, production APIs, domains, and recurring subscriptions require approval from the project lead and finance owner. Client-billed tools also require client approval before purchase or activation.

## Vendor Risk
Tools that process client data, personal data, credentials, payments, or production logs require security review. The review should consider access controls, encryption, data location, incident notice, support model, and deletion options.

## Ownership
Client-owned systems should be registered under client-controlled accounts whenever practical. If Code 5 Developers temporarily manages an account, ownership transfer must be documented before handover.

## License Management
Licenses must be tracked with owner, cost, renewal date, users, project, and cancellation instructions. Unused licenses should be removed during monthly project reviews.

## Open Source
Open-source packages may be used when licenses are compatible with the project and security risk is acceptable. High-risk packages, abandoned packages, or packages with unclear licenses require technical lead approval.
